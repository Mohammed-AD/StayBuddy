In Java AWT (Abstract Window Toolkit), layout managers are used to arrange components (buttons, text fields, labels, etc.) in a container such as a Frame, Panel, or Applet. Layout managers handle the positioning and sizing of components dynamically based on the container's size. AWT provides several built-in layout managers, each with its own way of arranging components.

1. FlowLayout

Description:

  • Arranges components horizontally in a row, from left to right.

  • If the container is not wide enough, it wraps components to the next row.

  • Aligns components in the center by default but can also be left or right-aligned.

Constructors:

java
FlowLayout()
FlowLayout(int align)
FlowLayout(int align, int hgap, int vgap)

  • align → FlowLayout.LEFT, FlowLayout.CENTER, FlowLayout.RIGHT

  • hgap → Horizontal gap between components.

  • vgap → Vertical gap between components.

Example:

java
import java.awt.*;
import java.awt.event.*;

public class FlowLayoutExample {
    public static void main(String[] args) {
        Frame f = new Frame("FlowLayout Example");
        f.setLayout(new FlowLayout());
        
        f.add(new Button("Button 1"));
        f.add(new Button("Button 2"));
        f.add(new Button("Button 3"));
        
        f.setSize(300, 200);
        f.setVisible(true);
    }
}

Event Delegation Model in Java

The Event Delegation Model (EDM) in Java is a mechanism used to handle events efficiently in GUI applications. It follows a publisher-subscriber pattern, where an event source (publisher) delegates the handling of events to a listener (subscriber). This approach improves performance and modularity compared to the older Event Handling Model, where components handled their own events.

How Event Delegation Model Works

  1. Event Source (Component) → The component (e.g., Button, TextField) generates an event when interacted with.

  2. Event Object → The generated event is encapsulated in an event object (e.g., ActionEvent, MouseEvent).

  3. Event Listener (Handler) → A separate object (listener) waits for and processes the event.

  4. Event Registration → The listener is registered with the event source using methods like addActionListener(), addMouseListener(), etc.

Steps to Implement the Event Delegation Model

  1. Implement a Listener Interface (e.g., ActionListener, MouseListener).

  2. Override the required method(s) to handle the event.

  3. Register the Listener with the event source using methods like addActionListener().

Types of Event Classes

Java provides a package called java.awt.event which contains predefined event classes. Some commonly used ones:

What is JDBC?

JDBC (Java Database Connectivity) is an API (Application Programming Interface) in Java that allows Java applications to interact with databases. It provides methods to connect to a database, execute queries, and retrieve results.

JDBC Components:

  1. JDBC Driver – Acts as a bridge between Java applications and the database.

  2. Connection Interface – Establishes a connection with the database.

  3. Statement Interface – Executes SQL queries.

  4. ResultSet Interface – Retrieves and processes results from queries.

Steps to Connect a Java Application with MySQL Database

1. Load the JDBC Driver

  • Use Class.forName("com.mysql.cj.jdbc.Driver").

2. Establish a Connection

  • Use DriverManager.getConnection(URL, USERNAME, PASSWORD).

  • Example MySQL URL:

    bash
    jdbc:mysql://localhost:3306/databasename

3. Create a Statement

  • Use Statement or PreparedStatement to execute SQL queries.

4. Execute the Query

  • Use executeQuery() for SELECT statements.

  • Use executeUpdate() for INSERT, UPDATE, or DELETE statements.

5. Process the Results

  • Retrieve data using ResultSet.

6. Close the Connection

  • Use close() method.

JDBC Application Code: Connecting to MySQL

The following Java program connects to a MySQL database and retrieves data from a table.

Requirements:

✔ MySQL Database installed
✔ MySQL JDBC Driver (mysql-connector-java.jar) added to the project

JDBC Code Example: Connecting to MySQL and Fetching Data

java
import java.sql.*;

public class JDBCExample {
    public static void main(String[] args) {
        // Database URL, username, and password
        String url = "jdbc:mysql://localhost:3306/testdb"; // Replace 'testdb' with your database name
        String user = "root"; // Replace with your MySQL username
        String password = "password"; // Replace with your MySQL password

        // SQL query
        String query = "SELECT * FROM students"; // Replace 'students' with your table name

        try {
            // Step 1: Load the MySQL JDBC Driver
            Class.forName("com.mysql.cj.jdbc.Driver");

            // Step 2: Establish Connection
            Connection con = DriverManager.getConnection(url, user, password);
            System.out.println("Connected to Database!");

            // Step 3: Create Statement
            Statement stmt = con.createStatement();

            // Step 4: Execute Query
            ResultSet rs = stmt.executeQuery(query);

            // Step 5: Process Results
            while (rs.next()) {
                int id = rs.getInt("id");
                String name = rs.getString("name");
                int age = rs.getInt("age");
                System.out.println("ID: " + id + ", Name: " + name + ", Age: " + age);
            }

            // Step 6: Close Resources
            rs.close();
            stmt.close();
            con.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

What is Connection Pooling in JDBC?

Connection Pooling is a technique used in Java Database Connectivity (JDBC) to manage database connections efficiently by reusing connections instead of creating new ones for each request.

When multiple users or processes need database access, opening and closing connections frequently can slow down performance. Connection pooling improves performance by maintaining a pool of pre-initialized connections that can be reused when needed.

How Connection Pooling Works

  1. A pool of connections is created when the application starts.

  2. When a request comes in, an existing connection from the pool is assigned to the request.

  3. After query execution, the connection is returned to the pool instead of being closed.

  4. The pool manages idle connections and removes unused ones to optimize resources.

Advantages of Connection Pooling

✅ Faster Performance – Reduces connection creation time.
✅ Efficient Resource Management – Limits the number of active database connections.
✅ Scalability – Handles multiple concurrent users efficiently.
✅ Reduced Memory Overhead – Prevents excessive memory usage by limiting open connections.

Implementing Connection Pooling in JDBC

1. Using Apache DBCP (Database Connection Pooling)

Apache DBCP (Commons DBCP) is a widely used library for connection pooling.

Steps to Implement DBCP Connection Pooling

  1. Add Apache Commons DBCP & Commons Pool dependencies (for Maven users):

    xml
    <dependency>
    <groupId>org.apache.commons</groupId>
    <artifactId>commons-dbcp2</artifactId>
    <version>2.9.0</version>
</dependency>

  2. Java Code to Implement Connection Pooling using DBCP

    java
    import java.sql.Connection;
import java.sql.ResultSet;
import java.sql.SQLException;
import java.sql.Statement;
import org.apache.commons.dbcp2.BasicDataSource;

public class DBCPConnectionPool {
    private static BasicDataSource dataSource = new BasicDataSource();

    static {
        // Configure the connection pool
        dataSource.setUrl("jdbc:mysql://localhost:3306/testdb");
        dataSource.setUsername("root");
        dataSource.setPassword("password");
        dataSource.setMinIdle(5);
        dataSource.setMaxIdle(10);
        dataSource.setMaxOpenPreparedStatements(100);
    }

    public static Connection getConnection() throws SQLException {
        return dataSource.getConnection();
    }

    public static void main(String[] args) {
        try (Connection con = DBCPConnectionPool.getConnection();
             Statement stmt = con.createStatement();
             ResultSet rs = stmt.executeQuery("SELECT * FROM students")) {

            while (rs.next()) {
                System.out.println("ID: " + rs.getInt("id") +
                                   ", Name: " + rs.getString("name") +
                                   ", Age: " + rs.getInt("age"));
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

Explanation of the Code:

  • A BasicDataSource object is created for managing a pool of connections.

  • setMinIdle(5) → Keeps at least 5 idle connections.

  • setMaxIdle(10) → Limits idle connections to 10.

  • getConnection() → Retrieves a connection from the pool.

  • The try-with-resources statement ensures automatic closing of resources.
